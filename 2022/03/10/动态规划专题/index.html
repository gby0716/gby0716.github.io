1、跳跃游戏

解法一：从前往后求解

    贪心思想，dp数组记录每个位置最远可以到达的位置。初始化，dp[0]=nums[0]

    递推公式：前一个位置最远能达到的位置和当前位置最远能到达位置中的较大值。

   dp[i] = max(i+nums[i], dp[i]);

    但是，前一个位置是否能到达当前位置呢？不一定！

    前一个位置能向后充分条件是dp[i-1] >= i，也就是前一个位置的能到达的最远位置必须不近与当前的位置。

    利用这个条件，可以提前终止遍历的过程。

    时间复杂度为o(n), 时间复杂度o(n), 需要一个和nums长度相同的dp数组

解法二：从后往前求解

    不断判断前一个位置的最远距离是否能达到当前目标。

    当前目标初始化为最后的下标：target = nums.length-1;

    如果存在nums数组元素满足nums[i] + i >= target，则表示在第i个位置可以到达当前的目标，从而只需要考虑在前面还未遍历到的位置是否存在能到达第i个位置的元素。

    因此如果存在满足nums[i] + i >= target的元素，可以将target更新为i, target = i; 直到i=0,停止遍历。

1、买卖股票的最佳时机 II

题目描述：

分析：题目中提到可以同一天购买和出售股票，但是同时最多只能拥有一支股票，所以在同一天可以不断购买和出售股票。

解题思路：因为可以在同一天不断购买和出售股票，如果假设我们拥有预知未来的能力，那么我们可以在价格较低的时候购入股票，等待下一次比购买股票时的价格高时出售股票。

    而出售股票之后，利用预知的能力，得到未来会有更高的股票价格，那我们就可以在同一天出售股票之后又购买股票，继续等到合适的时机出售股票。

    综上所述，利用预知的能力，我们可以在有高价的时候立刻出售股票，同时购买股票并继续等待更高的价格即可。例如：1，5，6 一次出售的最大利润为6-1，但是多次购买出售的最大利润为：5-1 + 6-1 结果是相同的。

2、最佳买卖股票时机含冷冻期

  题目描述：

解题思路：动态规划。首先将每天的状态简化为持有股票和未持有股票两种，为两种状态分别创建dp数组，dp数组中值的含义在持有股票时和为持有股票时的最大利润。

    根据这两种状态就可以简单地得到转移方程：

    今天持有股票：*have[i] = max(have[i-1], no[i-2]-prices[i])*，如果今天持有股票，那么这支持有的股票只有两个来源，1、昨天就持有的股票 2、在冷冻期结束之后，今天刚刚购买的股票

    今天未持有股票：no[i] = max(no[i-1], have[i-1]+prices[i]), 如果今天没有股票，那么没有持有的原因只有两种，1、昨天就没有，今天依旧没有 2、昨天持有，今天刚刚卖出

    通过上述的状态转移方程，很容易计算分别计算每天持有和未持有股票时的最大利润。不过需要注意的是，冷冻期之前至少还要有一天进行股票的买卖。

1、打家劫舍

题目描述：

分析：由于连续偷窃相邻的两个房间会触发报警器，所以相邻的房间不能都偷窃。所以最少需要间隔一个房间，但是如何确定偷窃哪些房间呢？由于小偷很聪明，所以他想了这样的办法来判断哪个房间值得偷窃：

    如果当前房间的价值加上间隔一个房间的价值如果大于相邻房间的价值，那就偷这个房间，否则就保留上一个房间偷窃后的结果。

**解题思路:**通过上述分析，很容易判断这个过程适合动态规划。假设有这样的场景，这个小偷背了一个背包装他盗窃的财物，从这条街的第一家开始偷窃直到最后一家，那么每家的财物价值就是nums数组的值，而

    小偷的背包可以当作一个dp数组，dp数组的每个值代表小偷路过一户人家时背包里的最大财物金额。

    所以dp数组的状态转移方程为：dp[i] = max(dp[i-2] + nums[i], dp[i-1]), 由于只有两个房间时偷窃哪个房间是容易确定的，所以dp数组的初始化为：*dp[0] = nums[0], dp[1]=max(nums[0], nums[1])*。

    （其实从后往前的过程是一样的，就是从街尾到街头）。
